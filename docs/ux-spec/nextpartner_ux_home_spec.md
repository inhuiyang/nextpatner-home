nextpartner_ux_home_specnextpartner_ux_home_specnextpartner_ux_home_spec# NextPartner 메인 페이지 UX 개선 스펙

> 대상: 50·60대 시니어 구직자  
> 목적: 사용성 개선 및 채용공고·프로젝트 유형 구분 명확화  
> 제약: 기존 디자인 시스템 유지 (컬러·폰트·버튼 스타일 변경 없음)

---

## 1. 개선 레이아웃 구조 (위 → 아래)

```
┌─────────────────────────────────────────────────┐
│  GNB: 상단 네비게이션 (개선)                      │
│  [로고] [채용] [프로젝트] [이력서] …              │
├─────────────────────────────────────────────────┤
│  Zone 0: 히어로 슬로건 (현행 유지, 로그인 전용)   │
│  "중장년 맞춤 일자리, NextPartner가…"            │
├─────────────────────────────────────────────────┤
│  Zone 1: 시니어 헤더 대시보드 (신규)              │  ← 최우선 노출, 이벤트 배너보다 상위
│  [인사말 + 아바타] [현황 카드 3종] [고객센터]     │
├─────────────────────────────────────────────────┤
│  Zone 1-B: 메인 이벤트 배너 (신규)               │  ← 대시보드 바로 아래
│  [이미지 + 슬라이드형 와이드 배너]                │
├─────────────────────────────────────────────────┤
│  Zone 2: 넥스트 액션 배너 (신규, 조건부 노출)     │
├─────────────────────────────────────────────────┤
│  Zone 3: 탭 분리 공고 섹션 (개선)                │
│  [정규직·계약직 채용] [프로젝트·프리랜서]         │
│  └─ 5-5. 이력서 CTA 섹션 (Zone 3 하단 포함)     │
├─────────────────────────────────────────────────┤
│  Zone 4: 구직에 도움이 되는 정보 (현행 유지)      │
│  [탭형 콘텐츠 카드 3열]                          │
├─────────────────────────────────────────────────┤
│  Zone 4-C: 일자리 관련 최신 기사 (현행 유지)      │
│  [2열 그리드, 기사 8건]                          │
├─────────────────────────────────────────────────┤
│  Zone 4-B: 띠 이벤트 배너 (신규)                 │  ← 푸터 바로 위
│  [텍스트 + 버튼 중심 전체 너비 띠]               │
└─────────────────────────────────────────────────┘
```

## 2. GNB — 상단 네비게이션

### 현행 구성
```
[로고] [채용공고] [프로젝트] [이력서]    [서비스 소개] [공지사항] [FAQ] [알림] [내 계정 ▾]
```

### 변경 사항
| 위치 | 현재 | 변경 후 | 비고 |
|---|---|---|---|
| 좌측 메뉴 1번 | 채용공고 | **채용** | Zone 3 탭과 통일, 공간 절약 |
| 좌측 메뉴 2번 | 프로젝트 | 프로젝트 | 유지 |
| 좌측 메뉴 3번 | 이력서 | 이력서 | 유지 |
| 우측 메뉴 전체 | 현행 유지 | 현행 유지 | — |

- GNB의 "채용" 클릭 시 → Zone 3의 "정규직·계약직 채용" 탭이 활성화된 채로 스크롤 이동
- GNB의 "프로젝트" 클릭 시 → Zone 3의 "프로젝트·프리랜서" 탭이 활성화된 채로 스크롤 이동

## 4. Zone 1 — 시니어 헤더 대시보드

### 4-1. 레이아웃
- 3-column 그리드: `[인사말] [현황 카드] [고객센터]`
- 비율: `2 : 5 : 3` (대략)
- 모바일(~768px): 현황 카드 2×2 그리드 → 고객센터 전체 너비로 하단 배치

### 4-2. 인사말 영역 (좌측)
```
[아바타 or 이니셜 원형]
홍길동님, 안녕하세요.
내 정보 한눈에 보기 >
```
- 이름 폰트: 기존 대비 +2px (가독성)
- "내 정보 한눈에 보기" → 마이페이지 링크

### 4-3. 현황 카드 3종 (중앙)
카드를 가로로 3개 배치. 각 카드 구성:
```
┌──────────────┐
│   지원 완료   │  ← 12px, muted
│      3건      │  ← 32px bold, 브랜드 컬러
│  이력서 보기  │  ← 12px 링크 텍스트
└──────────────┘
```

| 카드명 | 표시 내용 | 클릭 이동 |
|---|---|---|
| 지원 완료 | N건 | 내 지원현황 |
| 면접 예정 | N건 | 면접 일정 |
| 이력서 완성도 | N% | 이력서 편집 |

- 이력서 완성도 카드: 퍼센트 아래 얇은 프로그레스 바 추가 (기존 디자인 시스템 progress 컴포넌트 사용)
- 비로그인 상태: 카드 대신 "로그인하고 내 현황 확인하기" CTA 버튼 1개

### 4-4. 고객센터 박스 (우측)
```
┌───────────────────────┐
│ 📞 고객센터           │
│ 02-0713-7076          │  ← 20px bold
│ 평일 09:00–18:00      │  ← 13px muted
│ (점심 12:00–13:00)    │
│                       │
│ [전화 바로가기]        │  ← tel: 링크 버튼
│ [카카오 채널 문의]     │  ← 외부 링크
└───────────────────────┘
```
- 박스: 연한 배경(`#FAECE7`), 라운드 코너, 테두리 없음
- 전화번호: `<a href="tel:0207137076">` 처리 (모바일 자동 전화)
- 항상 노출 (로그인 여부 무관)

## 5. Zone 1-B — 메인 이벤트 배너

### 위치 및 우선순위
- Zone 1(대시보드) **바로 아래** 배치 — 대시보드가 항상 시각적 우선순위 우위
- 이벤트 없을 때: 섹션 자체 미노출 (공백 없이 Zone 2로 이어짐)

### 구조
```
┌────────────────────────────────────────────────────┐
│  [이벤트 이미지 영역]          [텍스트 + CTA 영역]  │
│  좌측 40% 이미지               우측 60%             │
│                                이벤트 제목 (20px)   │
│                                부제/설명 (14px)     │
│                                [자세히 보기] 버튼   │
└────────────────────────────────────────────────────┘
  ●  ○  ○   (페이지 인디케이터 — 복수 이벤트 시)
```
- 높이: 160–200px 고정
- 복수 이벤트: 자동 슬라이드 (5초 간격) + 좌우 화살표 + 인디케이터 dots
- 단일 이벤트: 슬라이드 없이 정적 노출, 인디케이터 미표시
- 모바일: 이미지 상단 / 텍스트 하단 2단 세로 배치, 높이 자동

### 데이터 구조
```json
{
  "id": "event_001",
  "imageUrl": "https://...",
  "imageAlt": "이벤트 이미지 설명",
  "title": "이벤트 제목",
  "description": "부제 또는 짧은 설명",
  "ctaLabel": "자세히 보기",
  "ctaUrl": "/events/001",
  "startDate": "2026-05-01",
  "endDate": "2026-05-31",
  "isActive": true
}
```
- `isActive: false` 또는 기간 외 항목은 자동 필터링
- CMS 또는 `GET /api/banners?type=main` 으로 목록 수신

## 6. Zone 2 — 넥스트 액션 배너 (조건부)

### 노출 우선순위 로직
```
if (이력서 완성도 < 80%) → "이력서를 완성하면 지원 확률이 높아져요!"
else if (면접 예정 있음) → "N일 후 면접이 있어요. 준비하세요!"
else if (미지원 추천공고 있음) → "회원님께 딱 맞는 새 공고 N건이 도착했어요."
else → 배너 미노출
```

### 배너 구조
```
[아이콘] [메시지 텍스트 14px]         [CTA 버튼]  [X]
```
- 배경: `#FAECE7` (브랜드 연한 오렌지)
- 텍스트 컬러: `#712B13`
- X 버튼: 클릭 시 `sessionStorage`에 저장하여 해당 세션 동안 미노출
- 전체 너비 (양쪽 여백만 맞춤)

## 7. Zone 3 — 채용공고·프로젝트 탭 분리

### 7-1. 탭 명칭 변경
| 현재 | 변경 후 |
|---|---|
| 채용공고 | 정규직·계약직 채용 |
| 프로젝트 | 프로젝트·프리랜서 |
| (없음) | 탭 하단 설명 1줄 추가 |

탭 하단 설명 예시:
- 채용 탭: "기업에 소속되어 일하는 정규·계약직 채용 공고입니다."
- 프로젝트 탭: "기간제 단기 업무, 프리랜서 의뢰 공고입니다."

### 7-2. 채용공고 카드 [C2]
좌측 컬러바로 유형 표시:
```
┌─────────────────────────────────────────┐
│▌ (주)모리아타운            마감 D-1 배지  │  ← 4px 오렌지 바
│  백엔드 개발자 (정직)                     │  ← 16px bold
│  [송파구] [풀타임] [주 5일]               │  ← 칩 최대 3개
│  회사내규                                │  ← 급여 (muted)
│                [채용 정보 보기]           │
└─────────────────────────────────────────┘
```
- 좌측 컬러바: 4px, `#E85D30` (브랜드 오렌지)
- 정보 위계: 회사명(12px muted) → 직무명(16px 500) → 칩 → 급여 → 버튼
- 지원완료 상태: 버튼 "지원완료" + disabled 처리

### 7-3. 프로젝트 카드 [C3]
```
┌─────────────────────────────────────────┐
│▌ [프로젝트] (주)모리아타운   D-day 배지  │  ← 4px 파란 바 + pill 뱃지
│  UX 리서치 단기 프로젝트                  │  ← 16px bold
│  [재택] [1개월] [예산 협의]              │  ← 다른 메타 항목
│  회사내규                                │
│                [프로젝트 정보 보기]       │
└─────────────────────────────────────────┘
```
- 좌측 컬러바: 4px, `#185FA5` (파란 계열) — 채용공고와 명확 구분
- 상단 "[프로젝트]" pill 뱃지: 파란 배경, 흰 텍스트
- 메타 칩: 근무지 / 기간 / 예산범위 (급여 대신 예산 표시)
- 프로젝트 탭에서만 이 카드 사용

### 7-4. 추천 공고 섹션 통합
- 현행 "찾아드린 기회" + "채용공고" 두 섹션 → **탭 내 "추천" 필터**로 통합
- 탭 안에 정렬 옵션: [추천순] [최신순] [마감임박순]
- 추천 공고에만 "⭐ 추천" 소형 배지 부착

### 7-5. 이력서 CTA 섹션 (Zone 3 하단 고정)
공고 목록 하단, "더보기" 버튼 아래에 위치.
```
┌──────────────────────────────────────────────────┐
│  처음 드는 일자리를 찾지 못하셨나요?              │
│  [새 이력서 생성]          [기존 이력서 수정]     │
└──────────────────────────────────────────────────┘
```
- 두 버튼 나란히 배치 (양쪽 정렬)
- "새 이력서 생성" → `/resume/new`
- "기존 이력서 수정" → `/resume/edit`
- 탭(채용/프로젝트) 전환과 무관하게 항상 노출

## 8. Zone 4 — 구직에 도움이 되는 정보

> 현행 유지. 아래 구조는 예시이며 실제 탭 명칭·카드 내용은 개발자가 수정.

### 구조
```
구직에 도움이 되는 정보                    [정보 더보기 →]
[전체] [인사이트1] [인사이트2] [팁1] [팁2] [모리아타운의 이야기]

┌────────────┐  ┌────────────┐  ┌────────────┐
│ 이미지 없음 │  │  썸네일    │  │ 이미지 없음 │
│            │  │  이미지    │  │            │
│ [팁1]      │  │ [팁1]      │  │ [팁1]      │
│ 보도자료 대량 │ │ 보도자료 대량│  │ 보도자료 대량│
│ 생성5      │  │ 생성5      │  │ 생성5      │
│ 26-02-22   │  │ 26-02-22   │  │ 26-02-22   │
```

- 카드: 3열 그리드, 1행 노출 (카드 3개)
- 카드 구성: 썸네일(선택) + 카테고리 태그 + 제목 + 날짜
- "정보 더보기 →": 섹션 우상단, 전체 목록 페이지 이동

## 10. Zone 4-B — 띠 이벤트 배너

### 위치
- Zone 4-C(일자리 관련 최신 기사) **바로 아래**, 푸터 **바로 위**
- 이벤트 없을 때: 미노출

### 구조
```
┌────────────────────────────────────────────────────────────────┐
│  🎉  [이벤트 제목 · 핵심 문구]              [지금 참여하기 →]  │
└────────────────────────────────────────────────────────────────┘
```
- 높이: 56–64px (1줄 고정)
- 전체 뷰포트 너비 (full-width), 내부 콘텐츠는 최대 너비(max-width) 맞춤
- 배경: 관리자가 CMS에서 지정 (기본값 `#E85D30` 브랜드 오렌지)
- 텍스트 컬러: 배경 대비에 따라 흰색 또는 어두운 색으로 자동 결정 (WCAG AA 기준)
- 복수 이벤트: 3초마다 페이드 전환 (애니메이션), 마우스 hover 시 일시정지
- X 버튼: 우측 끝, 클릭 시 `localStorage`에 저장 → 해당 이벤트 ID 기준 재노출 안 함

## 12. 데이터 연동 포인트

| 컴포넌트 | 필요 API |
|---|---|
| StatusCards | `GET /api/user/dashboard` → `{applied, interview, resumePercent}` |

| MainEventBanner | `GET /api/banners?type=main` → 배너 배열 |
| StripEventBanner | `GET /api/banners?type=strip` → 배너 배열 |
| JobCard / ProjectCard | 기존 공고 API 유지, `type: "job" \| "project"` 필드 확인 필요 |
| CustomerServiceBox | 정적 데이터 (전화번호, 운영시간) — 환경변수 또는 CMS |
| InfoSection | `GET /api/contents?type=info` → 탭·카드 목록 |
| NewsSection | `GET /api/news` → 기사 목록 (최신 8건) |
